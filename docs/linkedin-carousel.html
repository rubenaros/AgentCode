02 · v1 — Lo que no hacer

12 errores. 3 horas. $4.21.

El error más caro

"Le pedí al primer agente que scaffoldeara Next.js con create-next-app. El comando hizo git init fresco y me borró el remote a GitHub, el plan del repo, y todo el contexto. 5 minutos, $0.40 quemados, 0 PRs."

Lección: los scaffolders agresivos no se mezclan con repos preexistentes. El arquitecto scaffoldea; los agentes construyen encima.

03 · Lo que destilé

Las 3 lecciones que cambiaron todo.

i.
El arquitecto define contratos y scaffoldea. El agente nunca toca esos archivos. Lo creativo va a él; lo mecánico, a ti.
ii.
"No toques X" no es contrato — es una sugerencia que el LLM ignora cuando le conviene. Necesitas enforcement estructural: CODEOWNERS, branch protection, tests que fallen si lo viola.
iii.
Los costos reales son 3 a 5× las estimaciones single-call. Por el re-envío de contexto en agentes con muchos tool calls. Mide en OpenRouter, no estimes.

04 · v2 — Los 4 patrones

Lo que metí en el template antes de invocar un solo agente.

01

Template completo

Scaffold + configs + CI + .gitignore agresivo + ESLint tuneado. Los agentes ya no scaffoldean nada.

02

Contratos como tests ejecutables

Funciones property-based con fast-check. Si la implementación del agente no las pasa, CI rojo. No es markdown — es enforcement real.

03

CONSTITUTION.md + CODEOWNERS

Reglas duras + bloqueo estructural sobre src/domain/. Sin OpenSpec ni Spec-Kit — los probé y no encajan.

04

Cost routing por agente

Frontend en DeepSeek V3.2 (4× más barato). Lógica + QA en Kimi K2.6. Calidad indistinguible para UI mecánica.

05 · El patrón clave

Spec ejecutable, no markdown.

Una suite property-based parametrizada. Cualquier implementación de la interfaz debe pasarla — o CI rojo.

// tests/contracts/SchedulerPort.contract.ts
export function schedulerPortContract(makeSut: () => Factory) {
  describe('SchedulerPort', () => {
    it('cancel devuelve candidates FIFO por createdAt');
    it('cancel filtra candidates por serviceId');
    it('cancel filtra por ventana (fuera del slot NO matchea)');
  });
}

// tests/engine.test.ts (escrito por el agente Dev Motor)
schedulerPortContract(() => ({
  scheduler: new Scheduler(repo, clock), repo, clock
}));

El test es el contrato. El agente no puede "interpretarlo" mal — si su implementación no pasa, no se mergea.
