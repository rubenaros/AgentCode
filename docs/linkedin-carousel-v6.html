Concepto

El modelo razona,
el arnés actúa

El arnés es toda la infraestructura que rodea al modelo: el loop, las herramientas, la memoria, la planificación, la verificación y la entrega.

En Claude Code, 1,6% es lógica del modelo.
98,4% es arnés.

Mi opinión

Dos capas rinden.
El resto, depende.

Se justifican: verificación con CI y entrega autónoma. Baratas, y las aporta GitHub.

Condicionales: la memoria (se escribió, no vi que se leyera) y la planificación (mismo costo, más varianza).

Cuándo valen

Las capas pesadas, según el proyecto

  • Proyecto chico / descartable: no las uses. Son lastre.
  • Equipo / features grandes / larga vida: pagan.
